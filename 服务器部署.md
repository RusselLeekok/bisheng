镜像更新后，“覆盖部署”的流程是：本地重新构建/打包，上传服务器，服务器 `docker load` 覆盖同名镜像，然后重启服务。

**本地重新构建**

在本地项目根目录执行：

```powershell
cd .\bisheng

docker compose -f docker\docker-compose.yml `
  -f docker\docker-compose.local-build.yml `
  -p bisheng build backend frontend
```

这会重新生成同名镜像：

```text
bisheng-backend:local
bisheng-frontend:local
```

重新打包：

```powershell
docker save -o bisheng-images.tar bisheng-backend:local bisheng-frontend:local
```

然后上传新的 `bisheng-images.tar` 到服务器，覆盖旧文件。

**服务器覆盖镜像**

假设你上传到了 `/opt/bisheng/bisheng-images.tar`：

```bash
cd /opt/bisheng

docker compose --env-file .env \
  -f docker-compose.yml \
  -f docker-compose.deploy.yml \
  -p bisheng down

docker load -i bisheng-images.tar
```

`docker load` 会把同名同 tag 的镜像覆盖到本机 Docker 里。

然后重新启动：

```bash
# 二选一
docker compose --env-file .env \
  -f docker-compose.yml \
  -f docker-compose.deploy.yml \
  -p bisheng up -d

docker compose --env-file .env \
  -f docker-compose.yml \
  -f docker-compose.deploy.yml \
  -p bisheng up -d --force-recreate backend backend_worker frontend
```

查看状态：

```bash
docker compose --env-file .env \
  -f docker-compose.yml \
  -f docker-compose.deploy.yml \
  -p bisheng ps
```

看后端日志：

```bash
docker compose --env-file .env \
  -f docker-compose.yml \
  -f docker-compose.deploy.yml \
  -p bisheng logs -f backend
```

**更稳的覆盖方式**

也可以不先 `down`，先 `docker load`，再强制重建容器：

```bash
cd /opt/bisheng

docker load -i bisheng-images.tar

docker compose --env-file .env \
  -f docker-compose.yml \
  -f docker-compose.deploy.yml \
  -p bisheng up -d --force-recreate backend backend_worker frontend
```

这样 MySQL、Redis、Milvus、MinIO、ES 这些数据服务不会被停掉，只重启你更新的前后端服务。通常我更推荐这个。

建议你用这套：

```bash
cd /opt/bisheng

docker load -i bisheng-images.tar

docker compose --env-file .env \
  -f docker-compose.yml \
  -f docker-compose.deploy.yml \
  -p bisheng up -d --force-recreate backend backend_worker frontend
```

然后访问：

```text
http://服务器IP:3001
```

**仅部署前端**

如果这次只改了前端页面，不需要重新构建后端。可以只打前端镜像、只上传前端镜像、服务器只重启 `frontend` 服务。

本地只构建前端：

```powershell
cd .\bisheng

docker compose -f docker\docker-compose.yml `
  -f docker\docker-compose.local-build.yml `
  -p bisheng build frontend
```

只打包前端镜像：

```powershell
docker save -o bisheng-frontend.tar bisheng-frontend:local
```

上传到服务器后加载镜像：

```bash
cd /opt/bisheng

docker load -i bisheng-frontend.tar
```

然后只重建前端容器：

```bash
docker compose --env-file .env \
  -f docker-compose.yml \
  -f docker-compose.deploy.yml \
  -p bisheng up -d --force-recreate frontend
```

查看前端状态和日志：

```bash
docker compose --env-file .env \
  -f docker-compose.yml \
  -f docker-compose.deploy.yml \
  -p bisheng ps frontend

docker compose --env-file .env \
  -f docker-compose.yml \
  -f docker-compose.deploy.yml \
  -p bisheng logs -f frontend
```

这样不会重启 `backend`、`backend_worker`，也不会影响 MySQL、Redis、Milvus、MinIO、ES 这些服务。
